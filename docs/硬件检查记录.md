# 硬件检查记录

记录对象：桌面 `声语信使.pdf` 与 `声语信使网表.net`，对应仓库路径 `hardware/syxs_board/`。

## 结论

整板原理图总体可用于当前 4-FSK 项目，但软件引脚必须跟整板网表同步。当前仓库里的接收端和发送端调试工程仍分别面向面包板/黑丸调试阶段，直接烧到整板前需要改 ADC 与 PWM 引脚。

## 发送链路

- 整板发送信号从 `PA0` 进入 `U10A MCP6002`，不是调试工程里的 `PA8`。
- `U10A` 当前为非反相放大，`R31=47k`、`R30=47k`，增益约为 2。调试时如果 PWM 经低通前幅度过大，建议先把 `R31` 改为 `0R`，让前级接近跟随，避免前级削顶或运放压摆率问题。
- `U10B` 后级为二阶低通，`R32=R33=10k`，`C37=C38=1.8nF`，后面通过 `C40=1uF` 交流耦合到 `TX_1`。
- `TX_1` 作为功放输入前的调试点是有价值的；如果它同时接到 `PA4` 预留测试点，`PA4` 应配置为 Analog 或输入高阻，不要输出驱动。
- PAM8302 为 BTL 输出，喇叭两端接 `VO+ / VO-`，不能把任意一端接地测量或接单端负载。
- PAM8302 的 `/SD` 由 `R24=10k` 上拉到 `5V_SYS`，Q3 可拉低关断。若后续要 MCU 静音控制，需要确认 `AMP_MUTE` 是否实际连到 MCU GPIO。

## 接收链路

- 整板接收输出经 `R36=1k` 与 `C41=2.2nF` 到 `PC4`，所以整板软件应使用 `ADC1_IN14 / PC4`。
- 面包板接收工程当前使用 `PA1 / ADC1_IN1`，这是原型阶段配置，不是整板最终 ADC 引脚。
- 话筒和模拟链路使用独立模拟供电/滤波思路是正确的；OLED 刷屏造成噪声时，应优先降低 I2C 翻转频率、减少 OLED 刷新、让 OLED 供电回流远离话筒/运放输入回路。

## MCU 与接口

- 矩阵键盘使用 `PA15/PC10/PC11/PC12/PB5/PB4/PB3/PD2`，软件应启用 SWD-only 调试，释放 JTAG 相关脚，尤其注意 `PA15`、`PB3`。
- USB-C 供电 CC 下拉为 5.1k，作为设备取电方向合理。
- `5V_SYS`、`AVCC`、功放电源和数字地回流需要在 PCB 布局上分区。功放喇叭电流和 OLED I2C/供电回流不要经过话筒和虚拟中点附近。

## 软件待改项

- 整板发送端：把 `software/stm32_send` 从 `PA8/TIM1_CH1` 改为 `PA0` 对应的 PWM/定时器输出。
- 整板接收端：把 `software/stm32_receive` 从 `ADC_CHANNEL_1 / PA1` 改为 `ADC_CHANNEL_14 / PC4`。
- 完整协议阶段：加入多位数字或短信息帧格式、长度、校验和重发策略。
