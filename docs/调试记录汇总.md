# 调试记录汇总

> **资料状态：早期原型记录。** 本文统计的是一位数字帧和 PC 播放测试，不代表当前第二版板卡的 48 字符、RS+CRC、多模式固件。当前工程说明见 [`software/stm32_second_board/README.md`](../software/stm32_second_board/README.md)。

原始串口记录位于 `logs/serial/`，最新完整记录为 `调试记录2.txt`。

## 接收端算法

- 采样率：16 kHz。
- 码元时间：20 ms，单窗口 320 点。
- 每窗口先减均值去直流，再对 1500/2500/3500/4500 Hz 做 Goertzel 能量检测。
- 有效判决包含绝对能量门限、最大能量与次大能量比值，以及静音噪声底估计。
- 静音日志已改为约 1 秒输出一次 `idle`，防止有效接收状态被串口刷屏覆盖。

## 一位数字帧

当前帧格式：

`00 01 10 11` + `01 10` + `data0 data1` + `~data0 ~data1` + `10 11`

其中 `~data` 表示两 bit 反码，即 `bits ^ 0x03`。

## Python 播放测试

`调试记录2.txt` 使用桌面最新版 `voice test.py` 依次播放 0 到 9。日志统计结果：

| 事件 | 次数 |
| --- | --- |
| `digit_ok` | 28 |
| `start_fail` | 2 |

每个数字的成功次数：

| --- | --- |
| 0 | 3 |
| 1 | 3 |

| 3 | 3 |
| 4 | 3 |

两次失败都发生在起始帧附近，后续把起始帧从容易与前导末尾混淆的组合调整为 `01 10`，发送端和接收端代码已同步。

## 发送端串口命令

发送端工程支持两类串口输入：

- `00`、`01`、`10`、`11`：连续输出指定 4-FSK 测试频率，便于示波器和接收端单频验证。
- `s`：停止测试音。
- `0` 到 `9`：发送对应数字帧。

调试串口参数：115200 baud，8 data bits，no parity，1 stop bit。

## 下一步建议

- 上整板前先单独测 `TX_1`，确认低通后 1500/2500/3500/4500 Hz 幅度一致且无明显削顶。
- 接收端整板版本要先改 ADC 到 `PC4/ADC1_IN14`，再重新测静音噪声和四个单频能量。
- 若 OLED 仍引入噪声，优先降低刷新频率，只在状态变化时局部刷新。
